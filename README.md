# Побег в Перекрёсток

Офлайн-метроидвания для браузера: бежим в продуктовый магазин. Портретная ориентация,
управление одним пальцем, процедурные уровни, лёгкий файтинг и головоломки, эффект
тамаготчи (персонаж растёт и открывает способности) и ежедневная выдача промокодов.

## Стек
- **Phaser 3** — игровой движок (физика, тайлы, ввод, сцены)
- **TypeScript** — типобезопасность
- **Vite** — дев-сервер и сборка
- **vite-plugin-singlefile** — сборка в один самодостаточный `index.html` для офлайна

## Играть в браузере (телефон / ПК)

После настройки GitHub Pages игра доступна по ссылке:

**https://powerviking145-lgtm.github.io/pobeg/**

### Один раз включить Pages

1. Репозиторий на GitHub → **Settings** → **Pages**
2. **Source:** **GitHub Actions** (рекомендуется — сборка из `dist/` при каждом push)

   *или* **Deploy from branch** → ветка `main` → папка **`/docs`**

3. Подождите 1–3 минуты (вкладка **Actions** — зелёная галочка)
4. Откройте ссылку выше в Chrome/Safari на телефоне

> Корневой `index.html` в репозитории — только для разработки (`npm run dev`).
> В браузере без сборки он **не работает**. Играет собранный файл в `docs/` или `dist/`.

## Запуск локально

```bash
npm install
npm run dev      # http://localhost:5173
```

## Сборка (офлайн)

```bash
npm run build    # создаёт dist/index.html — ОДИН файл со всем внутри
```

`dist/index.html` можно скачать и открыть двойным кликом без интернета: весь код,
графика и звук встроены в файл. Прогресс и промокоды хранятся в `localStorage` браузера.

## Управление (один палец)
- **Веди палец влево/вправо** — идти
- **Свайп вверх** — прыжок (повторный флик в воздухе — двойной прыжок, когда открыт)
- **Свайп вниз** — рывок/спуск (когда открыт)
- **Короткий тап** — удар / взаимодействие (рычаги)

На ПК для отладки: стрелки/AD — движение, Space/Up — прыжок, Shift/Down — рывок,
F — удар, E — взаимодействие.

## Геймплей
- **Цель** — найти ключ и добежать до дверей Перекрёстка (выход) наверху уровня.
- **Тамаготчи / рост** — еда и побеждённые враги дают очки роста. Стадии: Кроха →
  Малыш → Подросток → Взрослый. Каждая стадия увеличивает персонажа и открывает
  способность (двойной прыжок, рывок, удар-разлом). Рост сохраняется между забегами.
- **Метроидвания-гейтинг** — основной путь проходится базовым движением (нельзя
  застрять), а способности открывают бонусные зоны с дополнительной едой.
- **Головоломка** — рычаг открывает ворота к спрятанной еде.
- **Промокоды** — за прохождение выдаётся один промокод в сутки (лимит по дате).
  Все коды видны в разделе «Мои промокоды» с копированием и отметкой «активирован».

## Промокоды: демо и реальный пул
Сейчас коды генерируются (`GeneratedProvider`, формат `PEREK-XXXX-XXXX`).
Чтобы подставить реальный список, переключите провайдер — см.
[`src/systems/PromoSystem.ts`](src/systems/PromoSystem.ts):

```ts
import { setPromoProvider, PoolProvider } from "./systems/PromoSystem";

setPromoProvider(new PoolProvider([
  { code: "PEREK-REAL-0001", reward: "Скидка 10%" },
  // ...
]));
```

## Структура
```
src/
  main.ts                 конфиг Phaser (портрет, сцены, физика)
  config.ts               константы (размеры, цвета, тайлы)
  types.ts                общие типы
  scenes/                 Boot, Menu, Game, UI, Win, Promo
  systems/                LevelGenerator, TouchInput, GrowthSystem,
                          PromoSystem, SaveSystem, Sound, textures, ui, rng
  entities/               Player, Enemy
  assets/                 нарисованный фон (встраивается в сборку)
```

## Графика
Фон — нарисованная картинка (`src/assets/bg_city.png`). Спрайты персонажа, врагов,
предметов и тайлов рисуются программно в [`src/systems/textures.ts`](src/systems/textures.ts)
(мультяшный стиль, чистая прозрачность). Любой спрайт можно заменить реальным PNG:
положите файл и загрузите его под тем же ключом в `BootScene` — плейсхолдер не создастся.
